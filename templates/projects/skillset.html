Soon after graduation and right after starting my first role out of university, I found myself scrambling to build my resume and a network of contacts whilst gaining valuable insight into the hiring process. I basically felt the need to be in a good position to obtain a job or start a career with a strong backing and portfolio at my disposal. Whilst perusing for better ways of managing the myriad of methods resumes are created and maintained, I came across JSON Resume.

JSON Resume's Homepage

JSON Resume is a fantastic idea. The basic premise involves collecting and formatting pieces of information commonly associated with resumes (anything from basic contact information to education and skills) into a JSON object. This object can then be used to generate PDFs or websites of a resume using a variety of themes, while at the same time, creating a standard for developers to maintain the aformentioned and oft-repeated information.

Some of JSON Resume's schema

Utilizing the standard's pre-built website templates, I was able to get a simplistic resume site up with little hassle, as well as maintain my information in order across my own personal resume and LinkedIn. One small component I did forsee using some visual enhancement was the "Skills" section of the standard. This stemmed primarily from both the nature of the industry (ergo, the focus on the raw aptitude of the candidate themselves, including the languages/frameworks/etc. they're able to work with), as well as my preference for the visual over the plainly textual.

With the basic concept in mind, I spent some time browsing the various basic vizualizations that the JavaScript library D3.js was used to create, landing finally on the Force Directed Graph, which I felt conveyed the information was most necessary in accordance with the set of skills a programmer posseses.

Example of a basic Force-Directed-Graph

With the gist of the final deliverable in mind, I set out wrangling D3.js and an example Force Directed Graph implementation together. The application primarily intakes a user's full JSON Resume object, converts the skills section into a more D3 friendly CSV format, and then allows for the user to re-enter that generated CSV to generate their graph. An example of that CSV output is shown below:

SkillSet converter output

Some things to note are the 'source' and 'target' columns, which are derived from the JSON resume. These determine where the overall points in the graph are demarcated. Wherein the JSON resume, the schema dictates a "name" and "keywords" array under that skill name, the converter uses the name as the root node, and the keywords as its children.

Example SkillSet output graph

In order to make the generated graph a bit more intuitive for viewers, I also implemented several visual tweaks to assist with overall expressiveness. Within the 'Skills' object in JSON Resume, a user can also specifiy their strength in that particular skill, ranging from "Beginner," "Intermediate," and "Advanced," which the converter then uses as a final output value for D3 to intake. That value is then used to modify the opacity of that node, with the more faded skills being the weaker of the set, and the more vibrant ones being the stronger of the user.
